# Hacking Prático na Web: Ataques, Mitigações e Desafios

## 📖 Sobre o Projeto

Este repositório foi desenvolvido como parte do laboratório **Hacking Prático na Web: Ataques, Mitigações e Desafios**, oferecido pela DIO (Digital Innovation One).

O curso proporcionou uma visão prática sobre as principais vulnerabilidades encontradas em aplicações web, apresentando técnicas de identificação, exploração em ambientes controlados e estratégias de mitigação utilizadas para aumentar a segurança dos sistemas.

---

## 🚀 Conteúdos Aprendidos

### SQL Injection

Durante o laboratório, foram estudados os conceitos relacionados à vulnerabilidade SQL Injection, compreendendo seus impactos e riscos para aplicações web. Também foi apresentada a utilização de **Prepared Statements (Consultas Preparadas)** em PHP como uma das principais técnicas de mitigação.

### Cross-Site Scripting (XSS)

Foi possível compreender o funcionamento dos ataques XSS e seus impactos sobre aplicações e usuários. Além disso, foram abordadas sete práticas de prevenção, incluindo validação de entradas, sanitização de conteúdo e implementação de políticas de segurança.

### Scanners de Vulnerabilidades

O curso apresentou a importância das ferramentas de análise automatizada para identificação de vulnerabilidades, demonstrando como esses recursos auxiliam no processo de avaliação e fortalecimento da segurança das aplicações.

### Testes Black Box

Também foram estudados os testes Black Box, metodologia que permite avaliar uma aplicação sem acesso ao código-fonte, simulando a visão de um possível atacante externo.

### Web Application Firewall (WAF)

Foram abordados os conceitos de Web Application Firewall, destacando sua importância como camada adicional de proteção contra ataques comuns direcionados a aplicações web.

### Frameworks e Desenvolvimento Seguro

O laboratório reforçou a importância da utilização de frameworks e boas práticas de desenvolvimento seguro para reduzir riscos e fortalecer a proteção das aplicações.

### OWASP ZAP

Durante as atividades práticas, foi possível conhecer o OWASP ZAP (Zed Attack Proxy), ferramenta amplamente utilizada para interceptação de requisições, análise de tráfego web e identificação de vulnerabilidades.

## 🎯 Conclusão

Este laboratório proporcionou uma experiência prática e enriquecedora sobre segurança de aplicações web, permitindo compreender vulnerabilidades comuns, técnicas de mitigação e ferramentas utilizadas por profissionais da área. Os conhecimentos adquiridos servem como base para aprofundar os estudos em Pentest Web, Segurança Ofensiva e Desenvolvimento Seguro.
